# 汉字抽测卡 Web 程序 - 设计文档

**日期**: 2026-03-16
**版本**: v1.1
**状态**: 设计中

---

## 1. 项目概述

### 1.1 目标

构建一个面向小学生的汉字抽测卡 Web 程序，帮助使用者每日校验汉字掌握情况。

### 1.2 核心特点

- **无需登录**: 本地运行，开箱即用
- **数据本地化**: 汉字数据和评测结果均使用 Markdown 文件存储
- **儿童友好**: 色彩丰富的界面，适合小学生使用
- **智能抽测**: 根据历史掌握情况动态调整抽测内容

## 2. 数据设计

### 2.1 汉字数据源

**文件位置**: `data/characters/*.md`

**格式示例**:

```markdown
# 一年级下册

## 第一课：春天来了

| 汉字 | 拼音 | 释义 | 例句 |
|------|------|------|------|
| 春 | chūn | 春季，一年的第一季 | 春天来了，花儿开了。 |
| 来 | lái | 从别的地方到说话人所在的地方 | 小明来到学校。 |

## 第二课：小蝌蚪找妈妈

| 汉字 | 拼音 | 释义 | 例句 |
|------|------|------|------|
| 蝌 | kē | 蝌蚪，蛙或蟾蜍的幼体 | 小蝌蚪在水里游来游去。 |
| 蚪 | dǒu | 见"蝌" | 小蚪蚪长大了变成青蛙。 |
```

**解析规则**:
- 文件编码：UTF-8
- 学期名称：一级标题（#）内容
- 课文名称：二级标题（##）内容，格式为 `第X课：课文名`
- 表格要求：
  - 必须包含表头行：汉字、拼音、释义、例句
  - 每行一个汉字，独立成项（多字词语拆分为多行）
  - 忽略空行和分隔符行（|---|---|）
  - 处理缺失值：空单元格使用空字符串
  - 额外列：允许存在，但会被忽略
  - 表头本地化：支持 "字/拼音/意思/例子" 等变体（通过关键词匹配）

**数据验证**:
- 汉字字段不能为空，且必须是单个 Unicode 汉字字符
- 拼音字段不能为空，不符合拼音格式的字标记为警告但仍可导入
- 无效行（缺少必填字段）将被跳过并记录警告日志

### 2.2 评测记录

**文件位置**: `data/records/YYYY-MM-DD.md`

**格式示例**:

```markdown
# 2026-03-16 评测记录

## 抽测配置
- **课文范围**: 第一课、第二课
- **抽测数量**: 20
- **抽测模式**: 混合（汉字→拼音 50%，拼音→汉字 50%）

## 评测结果

| 汉字 | 拼音 | 课文 | 模式 | 结果 | 时间 |
|------|------|------|------|------|------|
| 春 | chūn | 第一课 | 汉字→拼音 | 掌握 | 19:30:15 |
| 蝌 | kē | 第二课 | 拼音→汉字 | 未掌握 | 19:30:45 |
| 来 | lái | 第一课 | 汉字→拼音 | 模糊 | 19:31:02 |

## 统计
- 总数: 20
- 掌握: 15
- 模糊: 3
- 未掌握: 2
- 正确率: 75%
```

### 2.3 掌握状态追踪

系统根据历史记录计算每个汉字的当前掌握状态。

**状态定义**:

| 状态 | 定义 |
|------|------|
| **新字** | 该汉字从未出现在任何评测记录中 |
| **已掌握** | 最近 3 次抽测中至少 2 次标记为"掌握"，且最近一次不是"未掌握" |
| **模糊** | 最近一次抽测标记为"模糊"；或最近 3 次抽测结果为 1掌握+1模糊+1未掌握（混合状态）|
| **未掌握** | 最近一次抽测标记为"未掌握"；或抽测次数 < 3 且从未标记为"掌握" |

**计算规则**:
1. **时间窗口**：使用全部历史记录，不限制时间范围
2. **最近 3 次**：按时间倒序排列，取最近的 3 条记录
3. **少于 3 次**：按实际次数计算
4. **结果优先级**：未掌握 > 模糊 > 掌握（用于边界情况判定）

**示例**:
| 历史记录（从新到旧） | 计算结果 |
|---------------------|----------|
| 掌握、掌握、掌握 | 已掌握 |
| 掌握、掌握、未掌握 | 已掌握（2/3 掌握） |
| 模糊、掌握、掌握 | 模糊（最近一次模糊） |
| 未掌握、掌握、掌握 | 未掌握（最近一次未掌握） |
| 掌握、模糊、未掌握 | 模糊（混合状态） |
| 未掌握 | 未掌握（只有一次且未掌握） |

### 2.4 文件并发与锁定

**设计假设**：本系统为单用户本地使用，不考虑并发写入场景。

**文件写入策略**:
- 评测记录文件：按日期创建新文件，同一日期内的多次抽测追加到同一文件
- 写入模式：追加写入（append），避免文件锁复杂性
- 异常处理：写入失败时重试 3 次，仍失败则提示用户并保存到临时文件

**数据一致性**:
- 读取数据时，容忍最多 1 秒的数据延迟（文件系统缓存）
- 页面刷新后重新读取文件，确保显示最新数据

## 3. 功能设计

### 3.1 抽测引擎

#### 3.1.1 抽测流程

1. 用户选择课文范围（可多选）
2. 系统根据历史记录分析每个汉字的掌握状态
3. 按优先级抽取指定数量的汉字：
   - 第一优先级：当前标记为"未掌握"的字
   - 第二优先级：当前标记为"模糊"的字
   - 第三优先级：新字（从未抽测）
   - 第四优先级：已掌握的字（用于复习）
4. 随机分配抽测模式（汉字→拼音 或 拼音→汉字）

#### 3.1.2 每日抽测量

- 默认: 20 个/天
- 用户可配置: 10-50 个
- 如果某优先级的字不足，自动从下一优先级补充

#### 3.1.3 会话状态管理

**状态存储方式**: 服务端内存存储（适合单用户使用）

**会话生命周期**:
1. 用户点击"开始抽测" → 创建新会话
2. 会话 ID 通过 URL 参数传递（如 `/quiz?session=abc123`）
3. 页面刷新 → 保留当前进度，可继续抽测
4. 关闭浏览器/超时 30 分钟 → 会话失效

**会话数据结构**:
```python
class QuizSessionState:
    session_id: str           # 唯一标识
    created_at: datetime      # 创建时间
    current_index: int        # 当前抽测位置
    characters: List[Character]  # 抽测字列表
    records: List[QuizRecord]    # 已完成的评测记录
    completed: bool           # 是否已完成
```

**异常处理**:
- 会话不存在/过期：重定向到首页，提示"会话已过期，请重新开始"
- 重复提交同一字：以后提交为准，覆盖前一次结果

### 3.1.4 抽测交互

**汉字→拼音模式**:
1. 显示：大字展示汉字（字体大小 120px）
2. 隐藏：拼音、释义、例句
3. 用户回忆后点击"显示答案"
4. 显示完整信息后，用户标记：掌握/模糊/未掌握
5. 自动进入下一字，或显示"本次抽测完成"

**拼音→汉字模式**:
1. 显示：拼音（大号）+ 释义
2. 隐藏：汉字、例句
3. 用户回忆后点击"显示答案"
4. 显示完整信息后，用户标记：掌握/模糊/未掌握
5. 自动进入下一字，或显示"本次抽测完成"

**快捷键支持**（桌面端）：
- 空格键：显示答案
- 1/2/3：快速标记 掌握/模糊/未掌握
- ←/→：上一字/下一字（仅允许查看已完成的字）

### 3.2 错字本

- 汇总所有历史评测中标记为"未掌握"的汉字
- 显示每个字的未掌握次数和最近评测时间
- 支持一键将错字加入今日抽测
- 支持按课文筛选

### 3.3 打印功能

**打印范围选择**：
- 按课文：选择特定课文打印
- 错字本：打印所有未掌握的字
- 自定义：手动选择要打印的字

**卡片布局**：
- 纸张：A4 横向（297mm × 210mm）
- 卡片尺寸：90mm × 60mm（每页 3×3 = 9 张）
- 间距：卡片间 15mm，页边距 15mm

**卡片设计**：
- 正面（上半部分）：
  - 汉字模式：大号汉字（48px）居中
  - 拼音模式：拼音（36px）+ 小字释义
- 背面（下半部分）：
  - 汉字、拼音、释义、例句完整信息
  - 可沿中线折叠，实现正面对应背面的卡片效果

**技术实现**：
- 使用 CSS @media print 媒体查询
- 打印预览页：隐藏无关 UI，仅显示卡片网格
- 黑白打印优化：移除彩色背景，保留边框线
- 分页控制：使用 CSS page-break 避免卡片被截断

**打印流程**：
1. 用户选择打印范围
2. 预览页显示卡片布局
3. 点击"打印"调用浏览器打印对话框
4. 用户选择打印机或保存为 PDF

### 3.4 错误处理

**数据文件错误**：

| 错误类型 | 处理方式 |
|----------|----------|
| MD 文件不存在 | 显示友好的空状态提示，引导用户创建数据文件 |
| 文件编码错误 | 尝试用 UTF-8 读取，失败则提示"文件编码错误，请使用 UTF-8 编码" |
| 表格格式错误 | 跳过无效行，记录警告，继续解析有效数据 |
| 汉字数据无效 | 标记为警告，跳过该字，继续加载其他字 |
| 缺少必填字段 | 使用空字符串填充，记录警告 |

**用户输入验证**：

| 输入 | 验证规则 | 错误提示 |
|------|----------|----------|
| 抽测数量 | 10-50 整数 | "请输入 10-50 之间的数字" |
| 课文选择 | 至少选择 1 个 | "请至少选择一个课文" |
| 抽测数量 > 可用字数 | 自动调整为可用字数 | 提示"已调整为最大可用数量 X" |

**运行时错误**：
- 写入记录失败：重试 3 次，失败则保存到 `.tmp` 文件并提示用户
- 会话过期：重定向首页，提示重新开始
- 未找到汉字数据：显示引导页面，提供示例数据下载

### 3.5 统计看板（可选增强）

- 总汉字数 / 已掌握数 / 未掌握数
- 连续学习天数
- 最近 7 天正确率趋势
- 掌握进度条（按课文）

### 5.3 核心数据模型

```python
class Character:
    char: str           # 汉字
    pinyin: str         # 拼音
    meaning: str        # 释义
    example: str        # 例句
    lesson: str         # 所属课文
    semester: str       # 所属学期

class QuizRecord:
    char: str           # 汉字
    pinyin: str         # 拼音
    lesson: str         # 课文
    mode: QuizMode      # 抽测模式
    result: ResultType  # 结果：掌握/模糊/未掌握
    timestamp: datetime # 时间戳

class QuizSession:
    date: date          # 日期
    lessons: List[str]  # 课文范围
    count: int          # 抽测数量
    records: List[QuizRecord]  # 评测记录
```

## 6. API 设计

### 6.1 页面路由

| 路由 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 首页 |
| `/quiz` | GET | 抽测页 |
| `/result` | GET | 结果页 |
| `/mistakes` | GET | 错字本 |
| `/print` | GET | 打印页 |

### 6.2 API 详细定义

#### GET `/api/semesters`
**响应**：
```json
{
  "semesters": [
    {
      "id": "grade1-spring",
      "name": "一年级下册",
      "file": "grade1-spring.md",
      "total_chars": 200
    }
  ]
}
```

#### GET `/api/lessons?semester={semester_id}`
**参数**：
- `semester`: 学期 ID

**响应**：
```json
{
  "semester": "一年级下册",
  "lessons": [
    {
      "id": "lesson-1",
      "name": "第一课：春天来了",
      "char_count": 10,
      "mastered_count": 6
    }
  ]
}
```

#### GET `/api/characters?semester={id}&lessons={id1,id2}`
**参数**：
- `semester`: 学期 ID（可选，不提供则返回所有）
- `lessons`: 课文 ID 列表，逗号分隔（可选）

**响应**：
```json
{
  "characters": [
    {
      "char": "春",
      "pinyin": "chūn",
      "meaning": "春季，一年的第一季",
      "example": "春天来了，花儿开了。",
      "lesson": "第一课：春天来了",
      "mastery_status": "mastered"
    }
  ]
}
```

**请求体**：
```json
{
  "semester": "grade1-spring",
  "lessons": ["lesson-1", "lesson-2"],
  "count": 20,
  "mode_mix": 0.5
}
```

**响应**：
```json
{
  "session_id": "abc123def456",
  "total": 20,
  "characters": [
    {
      "index": 0,
      "char": "春",
      "pinyin": "chūn",
      "meaning": "...",

    }
  ]
}
```

```json

{

  }
```
```json
}
```
